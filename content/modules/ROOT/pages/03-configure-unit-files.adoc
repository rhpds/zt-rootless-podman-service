== Configure Unit Files

Now we’ll configure the `+httpd.container+` and `+httpd-data.volume+`
files.

== Edit the httpd.container file

Run the command below by clicking on the `+run+` button to configure the `+httpd.container+` file.

[source,bash,run]
----
tee -a /home/rhel/.config/containers/systemd/httpd.container << EOF
[Service]
Restart=always

[Container]
ContainerName=httpd
Image=docker.io/library/httpd
Label="io.containers.autoupdate=registry"
Environment=TZ=America/Vancouver
Environment=VERSION=docker
Volume=httpd-data.volume:/usr/local/apache2/htdocs:Z
PublishPort=8080:80/tcp

[Install]
WantedBy=default.target
EOF
----

The `+[Service]+` section specifies that the `+httpd.container+` service
should always be restarted regardless if it was shut down cleanly.

The `+[Container]+` section specifies the following: 1)
`+ContainerName+` is `+httpd+`. 2) `+Image+` defines where to pull the
image from. In this case, it is being pulled from the docker.io public
registry. 3) `+Volume+` defines that the directory inside of the
container `+/usr/local/apache2/htdocs+` should be exposed to the host so
that the user `+rhel+` can edit the contents. 4) `+PublishPort+`
defines port 8080 on the host should map to port 80 in the container.

== Edit the httpd-data.volume file

Run the command below by clicking on the `+run+` button to configure the `+httpd-data.volume+` file.

[source,bash,run]
----
tee -a /home/rhel/.config/containers/systemd/httpd-data.volume << EOF
[Volume]
User=rhel
Group=rhel
EOF
----

The `+Volume+` section specifies that the user `+rhel+` and members
of the group `+rhel+` have rights to modify the persistent volume
that will be created.

== Verify configuration files have been created successfully

Run the following command to find the unit files we just saved.

[source,bash,run]
----
ls /home/rhel/.config/containers/systemd/

Run the following to view the contents of `+httpd.container+`.

[source,bash,run]
----
cat /home/rhel/.config/containers/systemd/httpd.container
----

Run the following to view the contents of `+httpd-data.volume+`.

[source,bash,run]
----
cat /home/rhel/.config/containers/systemd/httpd-data.volume
----

If you were to create multiple container services, all the unit files
will be stored in `+/home/rhel/.config/containers/systemd/+`.
